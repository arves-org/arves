RCR-001 — Runtime v1.1 (additive, non-breaking)

=====================================================
  ARVES Runtime Change Request 001
  Target: Runtime v1.1   ·   Class: ADDITIVE / NON-BREAKING
  Status: RATIFIED (maintainer)   ·   Date: 2026-07
=====================================================

RCR-001 is the first Runtime Change Request processed against the frozen Runtime v1.0 (runtime-v1.0). It exists to discharge the documented v1.1 debt through the governed RCR path — the only permitted way the runtime changes — rather than via silent crate edits. It is deliberately scoped to changes that are additive and non-breaking; nothing here alters a public API shape, wire format, ContentId, or committed behaviour.

1. Motivation (closure-audit findings)

The independent, adversarial Build Program Closure Audit (see ARVES_BUILD_PROGRAM_CLOSURE.md and runtime/RUNTIME_FREEZE_v1.0.md, v1.1 backlog) recorded honest findings against the frozen runtime source. Four of them are pure hygiene / correctness debt that a new maintainer would trip over, and none require a breaking change:

  • #4 Stale crate headers — ~13 crates carried I1 skeleton — NO implementation yet headers even though arves-kernel and arves-persistence are fully implemented (working FileKernel/WAL/recovery/checkpoint; the workspace has 65 passing tests). A newcomer could not tell real crates from contract-only skeletons.
  • #5 CancellationToken::is_cancelled() no-op — arves-execution hard-returned false, so the Amendment-005 cooperative-cancellation primitive silently did nothing.
  • #6 Freeze-doc guarantee mis-scope — Engine Fabric / Capability Fabric were listed under "What v1.0 guarantees" while the exercised engine/capability logic actually flows through the SDK/Bridge in products/; the runtime crates are contract-only.
  • #7 Cargo.lock gitignored — a binary-producing workspace should commit its lockfile so clean clones resolve byte-identical pinned dependencies (a Determinism/Replay value).

Distributed I2–I6 remain deferred.

3. Non-breaking rationale

  • Headers (#4) and freeze doc (#6): comments / markdown only. Zero effect on compilation, types, or behaviour.
  • .gitignore (#7): a VCS-tracking change; it does not touch code or the build graph.
  • is_cancelled() (#5): the change is strictly additive.
  • The is_cancelled(&self) -> bool signature is unchanged; only its return value now reflects real state instead of a constant false.
  • Prior behaviour "a fresh, never-cancelled token reports false" is preserved — a new or Default token still reports false; the only new observable behaviour is that after cancel() it reports true, which is what Amendment-005 always specified.
  • The internal field changed from a _private: () placeholder to a private flag: Arc<AtomicBool>. The field was already private, CancellationToken is still Clone + Debug + Default, and no caller could name the old field — so no downstream code can break.
  • A new pub fn cancel(&self) is added. Adding a method is additive.
  • The crate remains std-only (Arc/AtomicBool are std); no new dependency.
  • The Cancellation, Executor, and OutcomeRouter traits are unchanged (still contract-only); no trait method was added, removed, or re-signatured.

4. Hard gate — cargo test must stay green

The v1.0 baseline is 65 passed / 0 failed (cargo test --manifest-path runtime/Cargo.toml --workspace). RCR-001 must keep it green and only add tests. After RCR-001 the workspace is 69 passed / 0 failed (65 baseline + 4 new arves-execution cancellation tests). No test was removed, weakened, or ignored. If any change here had broken the build or was not cleanly non-breaking, that change would have been reverted and left as documented debt.

5. Verification (run in-repo, offline)

cargo test --manifest-path runtime/Cargo.toml --workspace   # 69 passed; 0 failed
cargo build --manifest-path runtime/Cargo.toml --workspace  # clean
git check-ignore runtime/Cargo.lock                         # exit 1 (no longer ignored)

6. Governance trace

  • Instrument: RCR (per runtime/RUNTIME_FREEZE_v1.0.md §"Runtime Change Request").
  • Version bump: v1.0 → v1.1 (additive/backward-compatible minor).
  • Backlog items #4/#5/#6/#7 are marked ADDRESSED by RCR-001 in the freeze record.
  • IDR-006 respected: this is a Runtime-Team change to runtime/ (+ the root .gitignore), processed through the RCR path — never a product-side edit and never a spec change.
