# RCR-040 — Model ↔ Code conformance witness (link the checked TLA+ models to the running Rust) (Runtime v1.1)

**Status:** Applied (v1.1) · **Type:** additive, non-breaking (read-only introspection + a test)
· **Gate:** `cargo test --workspace` must stay green. Workspace **300 passed / 0 failed** with this
lane in the shared working tree (**+3** tests from this lane: the two conformance witnesses plus a
determinism guard; no prior test changed).

## Motivation

`verification/formal/` model-checks the **design**: TLC exhausts the abstract state machines of
`ARVES_Kernel.tla` (single-shard truth core) and `ARVES_Cluster.tla` (N-node per-shard Raft) and
proves the safety invariants over **all interleavings**. But nothing tied those proofs to the
**running reference code** — a certifier could re-run TLC and separately read the Rust, yet no
artifact asserted *"the code exhibits, on a concrete trace, exactly what the model proves."* That
disconnect — "model-check the DESIGN, not the Rust code" — is the last thing capping the Verification
dimension below its honest ceiling.

`ARVES_Kernel_Formal_Spec.md` §7 already names the intended bridge: **AKF-CS-1**, a byte-exact
conformance scenario tying the abstract model tokens to the reference kernel. This RCR implements
that bridge for the kernel model AND an analogous one for the cluster model.

Maintainer-directed hardening executed through the sanctioned RCR path — not a silent edit.

## Change (what shipped)

`runtime/crates/arves-kernel`:

- **`ClusterSim` — three read-only introspection accessors** (`src/cluster.rs`, the RCR-040 block),
  projecting the per-shard Raft group's OBSERVABLE state onto exactly the objects `ARVES_Cluster.tla`
  reasons about. They expose consensus **mechanism** metadata only (terms/indices), never Kernel
  truth (ORCH-001), and mutate nothing:
  - `log_terms_of(node, shard) -> Vec<u64>` — the model's `log[node]` term stamps in log order.
  - `leaders_by_term_of(shard) -> BTreeMap<u64, Vec<NodeId>>` — the model's `leaders_by_term`
    (delegates to the safety observer's `SimCluster::leaders_history`).
  - `committed_terms_of(shard) -> BTreeMap<u64, u64>` — the model's `committed` set as `idx -> term`
    (delegates to `SimCluster::committed_history`), single-valued per index iff State Machine Safety
    holds.
- **`tests/model_code_conformance.rs`** — the witness, 3 deterministic tests:
  1. `akf_cs1_kernel_model_conformance_witness` — runs the byte-exact **AKF-CS-1** trace on the real
     `RefKernel<MemWalStore>`, witnessing `OWN_001`, `ORCH_003_ReplayEquiv`, `ORCH_004`, and the §6
     order-sensitivity note. ContentIds recomputed from the runtime's OWN SHA-256
     (`arves-acs::content_id`); truth digest asserted byte-exact `0x7bb9b2e30ee7427c` (canonical) and
     `0xed1b740f02f0b8f4` (reordered) — the anti-drift tripwire ARVES_Kernel_Formal_Spec §8 Risk (b)
     calls for.
  2. `cluster_model_conformance_witness` — drives the real 3-node `ClusterKernel`/`ClusterSim`
     (`Server = {n1,n2,n3}`, matching `ARVES_Cluster_MC.cfg`) through a trace mirroring the model's
     actions (elect → commit → replicate → **term-changing failover** → converge), witnessing all
     five model safety invariants: `ElectionSafety`, `LogMatching`, `StateMachineSafety`,
     `LeaderCompleteness`, `LinearizableCommit`.
  3. `cluster_conformance_witness_is_deterministic` — the witness is a pure function of its seed
     (identical replica truth digests across two runs; ORCH-003).
- **`verification/formal/MODEL_CODE_CONFORMANCE.md`** (living evidence) — the full invariant → Rust
  assertion mapping table and the honest-scope statement.

**Additive & backward-compatible.** No existing behaviour, type, or signature changed; the frozen
`Kernel` trait is untouched; the three accessors are new read-only methods. **Zero new third-party
crates** (reuses `arves-acs` SHA-256, already a dep). No new dependency edge — the test crate uses
`arves-acs`/`arves-persistence`, both already `arves-kernel` deps, so **LAYER-001 is untouched**.

## Honest scope — what this IS and IS NOT

- **IS: a concrete-scenario conformance witness.** For ONE trace per model, every checked model
  invariant is mapped to a Rust assertion on the real code's observable state. "The model checks" and
  "the code behaves" become **linked** claims — the two can no longer drift silently (a change to
  either that breaks the mapped relationship fails the test).
- **IS NOT: a refinement proof / formal code proof.** It does not show the Rust refines the TLA+ over
  all states/interleavings. **TLC still owns the all-interleavings argument; this test owns the
  one-trace bridge.** The score claim must read "the model and the code are now linked by a running
  conformance witness **on the checked scenario**", not "the code is formally verified against the
  model".
- **IS NOT: new distributed guarantees.** It witnesses the EXISTING behaviour of the in-process
  simulation (`ClusterSim` honest scope unchanged: no network, no crash-durable raft state); it adds
  no host, no zero-trust, no GA claim.

## Verification

`cargo test --workspace` → **300 passed / 0 failed** in the shared working tree (**+3** from this
lane; every prior test still green). The new witnesses pass deterministically (fixed seeds, scripted
faults, injected ticks — no sleeps/wall-clocks/OS randomness).

The falsifiability of each mapped assertion is documented in MODEL_CODE_CONFORMANCE.md §6 (break
follower apply order → State Machine Safety witness fails; drop the §5.4.1 vote restriction →
LeaderCompleteness prefix check fails; change the `truth_hash` fold → AKF-CS-1 byte-exact check
fails).
