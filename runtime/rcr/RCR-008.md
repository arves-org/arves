# RCR-008 — Live L1 conformance: first executable ConformanceArtifact (Runtime v1.1)

**Status:** Applied (v1.1) · **Type:** additive (new module + bin + 2 downward deps; no existing
behavior/ABI change) · **Gate:** `cargo test --workspace` → **83 passed / 0 failed** (was 81; +2
live-conformance tests). Freeze baseline re-advanced via `freeze_check.py update`.

## Motivation (strategic-program rank 6; SYSTEM_GAP live-conformance / EVIDENCE_LEDGER §D)

The Scenario Conformance vocabulary in `arves-conformance` (Axis / Scenario / NodeProbe /
VerdictEngine / NodeEvidence / ConformanceArtifact / Verdict) was **typed but zero-instantiated** —
the single **L0** behaviour-conformance row in the whole Evidence Ledger, and one of only two
dimensions NOT capped by the G2 independence grade. A stranger had a byte target (ACS vectors) but
no runtime-BEHAVIOUR target.

**Discovery (corrects the plan's "freeze-clean" assumption):** those contract types are
`#[non_exhaustive]` with **no public constructors**, so a verification-side crate CANNOT instantiate
them. Populating the contract must therefore live INSIDE `arves-conformance` (the Conformance-stage
crate) — an additive RCR, exactly as RCR-006 added `property_check` there — not a freeze-clean
external crate. This is the constitutionally-correct home ("Implementation serves the Conformance
spec").

## Change (what shipped)

- **New module `arves-conformance::live`**: a concrete **`KernelProbe`** (impls the frozen
  `NodeProbe`) that drives a real `MemKernel` and **derives every check outcome from behaviour** —
  ORCH-004 (idempotent re-commit → `AlreadyCommitted`; content-integrity → `ContentIntegrity`,
  RCR-005), SHARD-001 (two-tenant, same-content/different-shard → distinct truths, no snapshot leak,
  RCR-007), ORCH-003 (recover from the WAL → identical `truth_hash`/count), OWN-001 (sole `commit`
  gateway). A **`LiveVerdictEngine`** (impls the frozen `VerdictEngine`) applies the Part-8
  worst-wins rule to emit the first live **`ConformanceArtifact`** for the **Core-Runtime** scenario.
  `run_core_runtime_scenario()` + `render()`.
- **New bin `conformance_live`**: runs it and prints the artifact; exits non-zero unless `Verdict::Pass`.
- **`Cargo.toml`**: `arves-conformance` gains `arves-kernel` + `arves-persistence` path deps —
  **downward** edges (rank 110 → 40 / 20), confirmed legal by the LAYER-001 architecture gate.
- **Tests**: `live_core_runtime_scenario_passes` (the real artifact PASSes; every invariant/property
  Held) and `verdict_engine_sinks_on_a_violated_invariant` (worst-wins: a synthetic Violated
  invariant → `Fail`).

## Scope (honest)

- **IS:** the FIRST live behaviour-conformance artifact, emitted by the real Kernel, drift-proof
  (wired into `evidence_probe.py`). An **Evidence-Level** raise (behaviour L0 → **L1 for the Kernel
  node**) at grade **G0/G1** — it proves the reference runtime *behaves*, it is **NOT** an
  independence-grade raise and NOT a G2 event.
- **IS NOT (ranks 7/8/13):** the full Information→Kernel→Query L1 triple. Only the **Kernel** node is
  observed live; the Connector (axis 1) and read-only Query projection are the next nodes, after
  which a full end-to-end scenario moves the whole 12-axis row. Additive; no public API changed.
