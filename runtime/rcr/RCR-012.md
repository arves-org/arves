# RCR-012 — Engine-enforced determinism & fabric-derived idempotency (v1.1)

**Status:** APPLIED (2026-07-05) · **Class:** additive (one new `InvokeError` variant;
one new crate edge, LAYER-001-legal) · **Closes:** `RUNTIME_FREEZE_v1.0.md` v1.1
backlog item **2** — *"Engine-enforced determinism — the fabric derives/enforces the
idempotency key rather than trusting an engine's self-declared `Determinism`."*

## The gap

Two trust holes in the Engine layer, both documented and both previously *trusted*:

1. **Self-minted keys.** `Engine::invoke`'s contract says the returned
   `Inference.key` MUST be the ORCH-004 content-addressable invocation key — but
   nothing verified it, and the reference `PureEngine` carried a documented
   **NON-CONFORMANT placeholder** returning `IdempotencyKey::default()` ("deriving
   the real key is a runtime-forward item").
2. **Self-declared determinism.** `Determinism::Deterministic` was *"a promise the
   engine makes to the fabric"* — never checked. A falsely-deterministic engine could
   poison replay/dedup (ORCH-003/004) silently.

## The change

**`arves-engine-fabric`** (+ downward dep `arves-acs`, rank 60 → 15, LAYER-001-legal):

- **`invocation_key(manifest, input)`** — the FABRIC's derivation of the ORCH-004 key:
  `scheme:name@version:<ACS-001 ContentId of input under domain 0x04 INVOCATION>`.
  Honest scope note: I1 read-sets are empty, so the read-snapshot component of the
  ORCH-004 triple is vacuous; it joins the derivation when Query reads land (I2+).
- **`invoke_enforced(engine, input)`** — (a) verifies the engine's returned key equals
  the fabric derivation (`FabricViolation::KeyMismatch`); (b) **double-invokes** a
  self-declared `Deterministic` engine and compares the two `Inference`s bit-for-bit
  (`FabricViolation::NondeterministicOutput`); Seeded/Nondeterministic engines are not
  re-invoked (recorded inference is authoritative on replay, ORCH-003) but are still
  key-verified. Honest scope: the double-invoke is a **probe, not a proof** —
  input-scoped or time-delayed nondeterminism can evade it (same boundary as the
  authoring kit's certify probe); the key check is exact.
- **`PureEngine::invoke`** now returns the fabric-derived key — the documented
  NON-CONFORMANT placeholder is closed; the crate STATUS header updated from
  "CONTRACT-ONLY / trivial placeholders" to "CONTRACT + FABRIC ENFORCEMENT".

**`arves-bridge`** — `invoke()` calls engines exclusively through `invoke_enforced`
(new `InvokeError::Fabric` variant), so enforcement lives on the REAL cognitive-chain
path: a mis-keyed or falsely-deterministic engine is refused **before any effect can
reach the Kernel** (proven: shard snapshot unchanged after refusal).

## Proof

- 6 new fabric tests: content-addressable key (same input ⇒ same key; re-version
  invalidates; embeds the ACS-001 invocation address); PureEngine returns the fabric
  key; conformant engine accepted; mis-keyed engine → `KeyMismatch`; a
  `Cell`-counter engine declaring `Deterministic` → `NondeterministicOutput`;
  declared-`Nondeterministic` varying output lawful (exactly 1 invocation consumed)
  with key still verified.
- 1 extended bridge test: the falsely-deterministic engine, *bound and authorized*,
  is refused at the fabric with **zero Kernel commits**.
- `cargo test --workspace` **91 → 97 / 0** (LAYER-001 architecture gate green over the
  new edge). Products unaffected: robustness 50/50, capstone 10/10 ✓, sound gate
  2× `SOUND-CERTIFIED (full ACS-001..005 surface)`.
