## Gap this closes (the honesty lever)

Through RCR-019..031 the entire I2‚ÄìI5 cluster ran on ONE vehicle: the in-process
deterministic MessageBus (`sim.rs`, a `VecDeque<Envelope>`). That is exactly the design's
mandated I2.1..I2.6 vehicle, but it forced every honesty note to read *"in-process
simulation, no network transport"*. This RCR upgrades the CLAIM from *"in-process
simulation"* to *"transport-agnostic, and exercised over a REAL OS socket"* ‚Äî the single
biggest reduction of the in-process caveat for the Runtime / thesis Ch4.

## The determinism argument (why real sockets do NOT break replay ‚Äî HARD RULE 4)

The consensus core (`raft.rs`) is a pure function of `(messages, seed, tick)`. A socket can
only reorder BYTES-in-flight; it cannot change WHICH messages exist. `TransportRound` sorts
every `drain` batch by its canonical `encode_envelope` bytes BEFORE the pure protocol
consumes it, so processing order is fixed by the harness, not the wire. Therefore:

> **RCR-032 invariant ‚Äî the transport moves bytes; the protocol decides truth; the harness
> fixes the order.**

The existing deterministic tests are untouched and stay bit-identical (`sim.rs` was not
modified; `InProcessTransport` is a faithful re-embodiment of its FIFO delivery, not a
replacement). The socket is a delivery layer UNDER the deterministic protocol ‚Äî never a
source of nondeterminism in committed truth.

## Design resolutions (recorded, never silently invented)

| # | Resolution |
|---|---|
| DR-1 | **The seam is a DELIVERY layer, not an ordering layer.** `Transport::drain` returns messages in UNSPECIFIED order; the harness canonicalizes. This is what lets a real socket (arrival-order delivery) sit under the pure protocol without threatening committed-truth determinism. |
| DR-2 | **`sim.rs` is NOT refactored to route through `Transport`.** The mandate is "existing deterministic bus implements UNCHANGED"; touching `SimCluster` risked changing digests of ~200 existing tests. Instead `InProcessTransport` re-embodies the same FIFO discipline behind the seam, and the equivalence proof runs InProcess-vs-Loopback through ONE identical driver (`TransportRound`) ‚Äî the transport is the only variable, which is a STRONGER isolation of transport-agnosticism than comparing against `SimCluster` would be. |
| DR-3 | **Loopback = one process, both endpoints local (127.0.0.1).** Genuinely crosses the OS socket boundary (real `write`/`read`, real framing, real partial reads) while staying a deterministic single-process TEST vehicle. Complete per-drain delivery is guaranteed by a sent/received frame counter (the transport owns all sockets, so it knows how many frames it wrote and reads until all arrive), turning a stuck socket into a loud panic, never a silent hang. |
| DR-4 | **Reconnect is IDLE-time.** The reset is injected at a quiescent boundary (`sent == received`, buffers empty ‚áí no frame in flight); the next `send` reconnects. This honestly proves "the transport survives a peer connection reset and re-establishes it" ‚Äî NOT mid-frame retransmission under loss (that needs real-latency/loss modeling, kept OQ). |
| DR-5 | **Wire codec is an internal deterministic framing, NOT the OQ-3 decision.** OQ-3 (the final protocol wire format ‚Äî dCBOR/ACS-002 canonical) stays open; this LE codec proves the SAME messages serialize/deserialize losslessly across a socket, which is what transport-agnosticism requires. |

## HONEST SCOPE ‚Äî what is proven vs what stays OQ

- **Proven (real, tested):** the consensus core is transport-agnostic; byte-identical
  committed truth over the in-memory bus AND over real length-framed TCP loopback sockets;
  the wire codec round-trips every message variant; framing survives partial reads; the
  round survives a full connection reset with transparent reconnect.
- **NOT proven / deferred (recorded, NEVER quietly claimed):** multi-HOST networking (this
  is loopback, one process); TLS / mutual authentication (design "new surface" OQ-7 ‚Äî mTLS
  is v2.0 debt); message loss / real latency / partition TIMING under a live network (the
  loopback delivers every sent frame; adversarial delivery stays the `sim.rs` filter/mangle
  model); a protocol-driven wire-format decision (OQ-3); crash-durable raft state.
- Read every "in-process simulation" claim in RCR-019..031 as **"in-process OR real-loopback
  transport (RCR-032)"** (a retroactive scope-annotation note was appended to RCR-019, 021,
  022, 024, 027, 031).
- **Anti-overread guard (permanent):** the proven property is **transport-agnosticism**
  (same bytes ‚Üí same committed truth across two delivery layers), which is guaranteed partly
  *by construction* ‚Äî the identical `TransportRound` harness canonicalizes every drain and
  loopback delivers every frame reliably, so the socket run is nearly forced to match. That
  is exactly what preserves determinism, and it is the honest ceiling of the claim. This
  score MUST stay worded "transport-agnostic / exercised over a real OS socket". It must
  **never** be upgraded to "network fault-tolerant", "resilient to partition/loss", or
  "distributed over the network" ‚Äî adversarial delivery (loss, latency, partition timing,
  multi-host, TLS) stays the `sim.rs` model + OQ, NOT this RCR.
