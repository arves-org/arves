# RCR-003 — Contract-crate traceability corrections (Runtime v1.1)

**Status:** Applied (v1.1) · **Type:** doc/citation corrections + one private-helper logic fix ·
**Gate:** `cargo test --workspace` must stay green → **75 passed / 0 failed** after this change
(was 71; +4 from the new `arves-invariants` Layer tests). Freeze baseline re-advanced via
`verification/freeze/freeze_check.py update` as part of this RCR.

## Motivation

The read-only Independent Architecture Review of the 11 CONTRACT-ONLY crates
(`verification/evidence/CONTRACT_CRATE_AUDIT.md`) found the frozen spec is **correct as written**;
the drift is in **`runtime/` crate citations** — four crates re-numbered or re-authored the
identifiers (IDR / ENG / CAP) they cite, and the `arves-invariants` `Layer` helper inverted one
edge — which would mislead a G2 party doing ID-based traceability against `runtime/` as a
reference. A runtime gap found during review is an RCR, never a silent edit (Freeze Record).

## Change (what shipped, per crate — doc-only unless noted)

- **arves-consensus** (RCR-CCA-01 / F-1,F-2,F-3): retagged the inverted IDR citations to the
  frozen IDR Batch 1 record — WAL / log-is-decision-trace / storage → **IDR-005** (convergence
  sentence noted as IDR-001's Engineering Refinement); joint-consensus **membership → IDR-003**
  (leader-election IDR-004 tags left correct); header/heading **"IDR-001..004" → "IDR-001..005"**.
- **arves-engine-fabric** (RCR-CCA-02..05 / F-4..F-7): dropped the crate's own conflicting
  ENG-001..005 numbering as authoritative and re-cited to the registered-normative invariants
  (ORCH-001/003/004, OWN-001, LAYER-001) + the Engine Graph Spec, noting the frozen ENG-00n
  numbering differs; scope notes that `EngineManifest.produces` maps to the spec's **Writes** (the
  distinct **Produces** artifact-type field is deferred) and that the manifest is an I1 **subset**
  (Preconditions/Failure/Retry/Timeout/Confidence/Cost/Latency deferred); reworded the `PureEngine`
  overclaim and marked its `IdempotencyKey::default()` body a **non-conformant placeholder**.
- **arves-capability-fabric** (RCR-CCA-06 / F-8): dropped the re-authored CAP-001..009 numbering
  and cited the registered invariants the crate actually upholds (ORCH-001/002/004, OWN-001,
  LAYER-001, SHARD-001); kept the loud "(proposed / non-binding / pending CCP-GATE)" framing.
- **arves-invariants** (RCR-CCA-07 / F-9,F-10 — the one **logic** fix): corrected the `Layer`

  permits the Engine→Query read) and **two tests** asserting both edge directions. Private-helper

- **arves-control-plane** (RCR-CCA-08 / C-1,C-2): retagged the misnumbered doc-comment IDR

- **arves-ontology** (RCR-CCA-09 / O-1,O-2): added a header citation to **ACS-004 §6.1/§8** noting
  the 18→8 `RootType` reconciliation and that the aspect field sets are the ratified ACS-004 binding.
- **arves-runtime** (RCR-CCA-10 / R-1): fixed `runtime/README.md`'s stale "I1 skeleton — interfaces

## Scope (honest — what this is and is NOT)

- **IS:** alignment of `runtime/` crate **citations/docs** to the frozen record, so a G2 reader is
  not misled — plus one private-helper (`Layer`) correctness fix with tests. No frozen **spec**
  (`standard/`, `spec-markdown/`, `corpus/`) was touched — the audit confirmed it correct.
- **IS NOT:** a behavior/ABI change. No public trait/type signature or struct field was renamed;
  `PureEngine::invoke`'s placeholder key is *documented*, not fixed (a follow-up); the Writes/Produces
  field rename and the real idempotency-key derivation are logged as runtime-forward items.
- **Independence unchanged: still G1.** This hardens `runtime/` as a citable reference; it is not a
  G2 event.
