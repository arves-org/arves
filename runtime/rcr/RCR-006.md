# RCR-006 — PropertyCheck / Suite: invariant → executable-proof catalog (Runtime v1.1)

**Status:** Applied (v1.1) · **Type:** additive (new lib module + shared-checker refactor of the
architecture-gate test; no behavior/ABI change) · **Gate:** `cargo test --workspace` → **80 passed
/ 0 failed** (was 78; +2 property-check tests). Freeze baseline re-advanced via `freeze_check.py update`.

## Motivation (SYSTEM_GAP_ANALYSIS #18)

The registered invariants were proven **ad-hoc**: LAYER-001/OWN-001 by
`tests/architecture_gate.rs`, ORCH-003/004 by `arves-kernel` behaviour tests — with **no single
artifact** binding each `InvariantId` to *its* proof or reporting coverage. "Invariant coverage
100%" was asserted, not checkable. RCR-006 makes it a catalog.

## Change (what shipped)

- New lib module **`arves-conformance::property_check`** — the invariant → proof catalog:
  - `ProofKind` = `InProcess { pass, detail }` | `CitedTest { location }` | `Pending { reason }`.
  - `catalog()` binds each of the 7 registered invariants:
    - **LAYER-001, OWN-001** → `InProcess`, **executed live** over the real Cargo graph
      (downward-only edges; exactly one `pub trait Kernel` owner).
    - **ORCH-004, ORCH-003, SHARD-001** → `CitedTest`, pointing at the biting Kernel tests
      (`walking_skeleton` behaviour_2/3/4/7, RCR-005 content-integrity; per-shard WAL keying).
    - **ORCH-001, ORCH-002** → `Pending` (arves-control-plane is contract-only, I2+ — an
      **honest** coverage gap, not a faked pass).
  - `run_suite()` → `(all_in_process_pass, proven_count, total)` for CI + coverage reporting.
- **Shared-checker refactor:** the pure LAYER-001/OWN-001 checkers (`layer_rank`,
  `workspace_edges`, `parse_internal_deps`, `layering_violations`, `truth_gateway_owners`) moved
  from `tests/architecture_gate.rs` into the lib module; the gate test now imports them, so the
  gate and the catalog share **one** implementation (no duplication, no drift).
- Tests: `property_check_suite_holds` (in-process proofs pass; coverage map is exactly 5 proven /
  2 pending / 7 total — a drift gate on coverage) + `layer_checker_bites` (the checker still flags
  an upward edge). The 4 architecture-gate tests are unchanged in behaviour.

## Scope (honest)

- **IS:** a checkable invariant→proof catalog; LAYER-001/OWN-001 executed in-process; the Kernel
  invariants cited to their tests; a live coverage figure (5/7 proven) that **cannot silently
  drift** (the suite test pins it).
- **IS NOT:** new *proofs*. It binds and reports EXISTING proofs; it does not prove ORCH-001/002
  (Control Plane not yet implemented — correctly Pending) and does not re-run cross-crate Kernel
  tests in-process (they are cited). Additive; no public trait/type/signature changed.
- **Independence unchanged: G1.**
