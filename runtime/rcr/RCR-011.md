# RCR-011 — Bridge request-id correlation (v1.1)

**Status:** APPLIED (2026-07-05) · **Class:** additive, backward-compatible (v1.1)
**Closes:** `RUNTIME_FREEZE_v1.0.md` v1.1 backlog item **1** — *"Bridge request-id
correlation — replace positional FIFO with explicit request ids."*

## The gap (product need)

The `arves-bridge` line protocol matched responses to requests **by position**: the
client kept a FIFO of pending callers and resolved the oldest on every response line.
One dropped, injected, or reordered line therefore shifted **every subsequent response
onto the wrong caller** — a silent cross-caller identity mix-up in the exact seam whose
job is identity. Input sanitization (no newlines, single-token capability) closed the
*reachable* desync paths, but the correlation itself remained positional debt.

## The change (additive)

**Server** (`crates/arves-bridge/src/bin/arves-bridge.rs`):
- A request line MAY carry `id=<token>` (1..=64 non-whitespace bytes) as its first
  token. The token is echoed verbatim as the first token of that request's response —
  on success **and** on `ERR` responses.
- A malformed id (`id=` empty, or > 64 bytes) is answered `ERR bad-id` **without an
  echo** — an untrusted token is never reflected back into the stream.
- Lines without the prefix behave byte-for-byte as before (backward compatible).
- Honest boundary: `ERR too-large` fires *before* parsing (bounded-line read), so it
  never carries an id; clients keep a FIFO fallback for id-less lines.
- The per-line protocol logic was extracted from `main` into `split_request_id` /
  `handle_request` / `respond`, so what is unit-tested is exactly what the server runs.

**Client** (`products/arves-sdk-ts/src/bridge.mjs` — living, product-side):
- Sends `id=r<N>` on every request; resolves responses **by id** (Map), with the FIFO
  fallback for id-less lines (pre-parse errors / older bridge binaries).
- Constructor gains an `args` option (test seam; default `[]`, no behaviour change).

## Proof

- 4 new server protocol tests (`rcr011_*`): plain lines unchanged; id echoed on
  success/error/idempotent-recommit; invoke-with-id; malformed-id refused without echo;
  MAX_ID boundary. `cargo test --workspace` **87 → 91 / 0**.
- Biting client regression (`products/robustness.test.mjs`, 49 → **50/50**): a fake
  bridge (`arves-sdk-ts/test/fake-reorder-bridge.mjs`) answers the first two requests
  in **reverse** order — with id-correlation each caller still receives its own
  response (`r1.index==1 && r2.index==2`); under the old FIFO matching the callers
  were silently swapped.
- End-to-end: capstone (`organization-day.capstone.mjs`) PASS over the new protocol;
  certification harnesses unaffected (they drive the address path whose first output
  token is still the ContentId when no id is sent — `verify_runtime_sound.py` green).
