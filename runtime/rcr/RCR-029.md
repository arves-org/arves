## HONEST LANGUAGE (load-bearing)

- The "agents" of this stage are **deterministic test actors, NOT AI models**.
- An agent identity is an **addressable registration the runtime can attribute
  actions to** ‚Äî NOT cryptographic authentication (v2.0 debt #8 / design OQ-1).
  ENFORCED (structural): the worn identity must exist as committed truth in the
  target shard before attribution; the attribution rides INSIDE the committed
  payload (the WAL trail carries Who on every replica); identity is
  content-addressed and can never denote two definitions (ORCH-004 + RCR-005).
  NOT ENFORCED (pinned by test, not hidden): that the CALLER is the agent ‚Äî any
  in-process caller can wear any registered identity (trusted-single-host model).
- The LCW `WorldView` is the working-memory **hydration surface** over committed
  truth, NOT a second external read API (Query keeps that; QUERY-001 stays
  PROPOSED). It is derived, disposable, never authoritative.

## Design resolutions (simplest option consistent with IDR-001..005 ‚Äî recorded, never silently invented)

| # | Resolution |
|---|---|
| DR-1 | **Code homes = the design's Q1 crates:** the shared-truth surface lands additively in `arves-lcw` (contract-only ‚Üí first behaviour, exactly the design ¬ß6.2 row-2 RCR); identity registration/attribution lands additively in `arves-control-plane` (the Control Plane REGISTERS and ATTRIBUTES ‚Äî it decides to attempt the write ‚Äî while the identity TRUTH is the Kernel's, ORCH-001; the RCR-027 DR-1 "behaviour beside the contract-only crate" pattern). No new crate, no new layer (Rule 3). |
| DR-2 | **"Built on I3's projections" = semantics, not a crate edge:** a literal `arves-lcw ‚Üí arves-query` dependency would be an UPWARD LAYER-001 edge (lcw 50 < query 60, ranks checked in `property_check.rs` first), so `WorldView` folds the WAL directly through `arves-persistence` (50 ‚Üí 20, downward) using the SAME deterministic replay-fold discipline RCR-023's `ShardProjection` proved, and the SAME digest tuple basis as the Kernel's `truth_hash` (equality proven by test). |
| DR-3 | **WorldView ownership honesty (OWN-001 vs the frozen LCW crate doc "LCW does not serve as a read API"):** the view is framed and built as the Layer-Matrix LCW row's rebuild input ("reads Kernel truth, events") ‚Äî the working-memory hydration surface ‚Äî NOT an external read API; Query's ownership of external read projections is untouched. It is derived + disposable and has no write surface. |
| DR-4 | **Single-owner rule made enforceable via ONE additive `#[non_exhaustive]` variant:** the frozen `LiveWorkspace` doc already states "at most one active live view per shard"; enforcing it needs a refusal the frozen `LcwError` lacked. `AlreadyOpen` is added (the exact purpose of `#[non_exhaustive]`); ownership transfer itself is Rust move semantics; `discard` releases the slot. The workspace retains NO live state. |
| DR-5 | **`PutCondition::IfAbsent` skip = `Ok(current revision)`, no mutation:** the frozen contract says the write "is skipped" without naming a return; reporting the existing revision (rather than an error) is the simplest reading that keeps the skip observable. Pinned by test. |
| DR-6 | **OQ-2 (agent-registry placement) resolved TRUTH-SIDE for this stage** ‚Äî the design's own lean (¬ß3.10 recovery argument): a registration is a committed, content-addressed truth in the agent's shard; it recovers with the truth base and never depends on orchestrator memory (proven: addressable from every replica after leader commit). Capability-Fabric-style registry state was the alternative; the truth-side reading needs zero new state owners. The full OQ-2 IDR remains open for later stages (e.g. definition supersession/revocation ‚Äî NOT built, see honest scope). |
| DR-7 | **No new ACS-001 domain tag:** the domain-tag registry is frozen `standard/` surface. Agent registrations and attribution envelopes are committed payloads, so they hash under the existing `COMMIT_CONTENT` tag; their self-describing payload MAGIC (`ARVES.AGENT.DEF.v1` / `ARVES.AGENT.ATTR.v1`) is a runtime-internal reference encoding, NOT a registered `uci.*` ontology type ‚Äî the O-006 CCP (design ¬ß6.2) remains the instrument for that and is NOT pre-empted here. |
| DR-8 | **Shard inside the identity body (SHARD-001 "shard-bound for life", ¬ß4 row):** the registration shard's `(tenant, workspace)` is part of the canonically-encoded, hashed definition body, so the SAME definition in two shards yields two DISTINCT `AgentId`s and no cross-tenant identity can exist even by hash collision of intent. |
| DR-9 | **Attribution gate basis = the caller-supplied `WorldView` (declared read basis):** the structural check runs against COMMITTED truth at an explicit version ‚Äî the runtime-grade elevation of the G1 caveat (in-memory maps) ‚Äî and staleness can only make the gate MORE restrictive because identities are immutable truths with no revocation in v1.x (recorded honest limit). The full OQ-3 serialization-point re-check (leader-side policy/conflict) is NOT built here ‚Äî it stays an I5 later-stage IDR obligation. |
| DR-10 | **`Revision` semantics:** first write of a cell = `Revision(1)` (`ZERO.next()`), matching the frozen doc's "revision of a cell that has never been written" = `ZERO`. Local mutation ordering only; never presented as a Kernel version (frozen doc). |

## Amendments & errata (adversarial revision, 2026-07-06 ‚Äî findings applied, never softened)

| # | Severity | Finding ‚Üí correction |
|---|---|---|
| **A1** | major | **SHARD-001 structural-gate hole in `is_registered` CONFIRMED and CLOSED.** The frozen `Kernel::commit` does not verify `content == ACS-hash(payload)` (RCR-005 admission only rejects same-address/different-payload forks), so a caller could lawfully commit a shard-B-bodied canonical definition INTO shard A's WAL ‚Äî and `is_registered`, which recomputed the id from the shard DECODED OUT OF THE PAYLOAD, would then admit shard B's identity in shard A, falsifying the advertised "addressable only through its own shard's world" / "cross-shard attribution refused" claims. Fix: `is_registered` now ALSO requires the decoded `(tenant, workspace)` to equal `world.shard()` ‚Äî the missing half of its own "defense in depth". Pinned by the new negative test `smuggled_foreign_shard_definition_is_refused_shard001` (commits the smuggled record through the raw gateway, asserts `is_registered`/`find_agent`/`propose_attributed` all refuse). |
| **A2** | minor | **Honesty overclaim scoped.** "Fabricated identities never enter the trail" holds only for the `propose_attributed` path: a hand-crafted `ATTR_MAGIC` envelope committed directly through the raw frozen gateway can carry an arbitrary/unregistered claimed Who, and `attribution_of`/`attributed_effects` decode and report that CLAIMED Who without an `is_registered` cross-check or ACS-001 34-byte length validation. The claim is now scoped ("via `propose_attributed`") in this record and in the honest-limit test; both readers carry an explicit HONEST-LIMIT doc note directing audit consumers needing registered-Who assurance to re-check `is_registered(world, &who)`. Reader hardening (a strict variant flagging non-registered attributions) is left as a recorded later-stage option, not silently pre-empted. |
| **E3** | minor | **`WorldView::hydrate_into` partial-write-on-error documented.** The method writes cells in commit order and returns the first `put` error with NO rollback, leaving a generic `WorkingMemory` partially hydrated. Impossible with `MemWorkingMemory` (`PutCondition::Always` cannot fail there), but the method is generic; the doc comment now says so out loud and instructs foreign implementations to treat any `Err` as "discard and re-hydrate", never a usable world (lossless-or-loud house style). Doc-level erratum only ‚Äî no behaviour change. |

## Invariants (registered only)

- **OWN-001** ‚Äî one owner per state, proven at the new surfaces: working memory
  has exactly one live view per shard (enforced `AlreadyOpen`); the world view is
  derived and cannot write; identities live ONLY in the truth base (drop the
  module's values, truth unchanged).
- **LAYER-001** ‚Äî two new edges, BOTH downward, ranks checked in
  `property_check.rs` BEFORE adding: lcw(50)‚Üípersistence(20),
  control-plane(90)‚Üílcw(50); lcw‚Üíquery deliberately absent (would be upward).
  Architecture gate green over the workspace.
- **SHARD-001** ‚Äî identity shard-bound for life (hashed shard in the body);
  cross-shard attribution refused; foreign-shard hydration refused; world views
  are per-shard folds that structurally never mix records.
- **ORCH-001** ‚Äî the Control Plane's new module owns NO truth: registration and
  attribution become state only through the frozen `Kernel::commit`; refusals
  commit nothing (proven by `committed_count`).
- **ORCH-002** ‚Äî nothing added is persistent orchestrator state: the workspace
  retains no live state; working memory is disposable and rebuilt from truth
  (discard + re-hydrate proof).
- **ORCH-003** ‚Äî the world view is a pure fold of the recorded trace: identical
  across re-reads, replicas and later commits at a fixed version; digest equals
  the Kernel's truth basis.
- **ORCH-004** ‚Äî identity and attribution are content-addressed: re-registration
  and duplicate effects converge to the same `TruthRef`, never fork; the id can
  never denote two definitions (RCR-005 at the gateway).
