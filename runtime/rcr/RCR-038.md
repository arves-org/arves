## Gap this closes (the honesty lever)

RCR-032 upgraded the transport claim from *"in-process simulation"* to *"exercised over a REAL OS
socket"* — but its `LoopbackTransport` lives entirely in ONE process (both socket endpoints local),
so the standing caveat was **"loopback, one process — not multi-host"**. This RCR reduces that
caveat one honest notch: from *loopback-in-one-process* to **cross-PROCESS networked consensus** —
genuinely separate OS processes, each hosting one raft node, exchanging every RPC over real
length-framed TCP, electing a leader and committing a write, and committing the **byte-identical
outcome** the in-process run commits.

It does **not** claim multi-HOST/WAN or hostile-network tolerance — those stay recorded OQ (below).

## The determinism argument (why real processes + real timing do NOT break replay — HARD RULE 4)

The raft core (`raft.rs`) is a pure function of `(messages, seed, tick)`. In the cross-process run
`tick` is driven by a wall-clock timer and the election timeout is real, so **WHO leads and under
WHICH term vary run-to-run** — that is *liveness/leadership*, and it is never allowed to feed
committed-truth CONTENT. The single quantity the proof compares, `outcome_content_digest`, is a pure
function of the *proposed* outcome's address+payload; it is invariant across leader, term, and
timing. Therefore whichever process wins the real election, the committed OUTCOME content is
byte-identical to the in-process run.

> **RCR-038 invariant — real sockets and a real election move liveness; the committed outcome
> content is a function of the proposal, never of the wire or the clock.**

### What the digest equality does and does NOT prove (anti-coincidence honesty)

`outcome_content_digest` is a pure hash of the FIXED proposal string (`TAG="payload"`), so for any
successful commit of that proposal the digest is byte-identical **by construction** — the digest
equality by itself does NOT independently distinguish genuine consensus from coincidence, and this
RCR does not claim it does. The digest is used only to show the committed CONTENT is timing- and
leader-independent (not corrupted/reordered by the wire), NOT to prove that agreement happened.

The **load-bearing consensus evidence** is the separate assertion that a **quorum advanced
`commit_index >= 1` over real TCP**: a follower can only advance its commit index after receiving
`AppendEntries` (log replication + `leader_commit`) from the elected leader across a real socket.
The `multiprocess` test asserts exactly this — `committed.len() >= 2` where each committed process
reported `COMMIT_INDEX >= 1` — so it FAILS if real TCP consensus did not occur (no quorum, no
replication ⇒ fewer than two processes reach `commit_index >= 1`). The digest-equality check is the
*content-integrity* layer stacked on top of that consensus evidence, not a substitute for it.

The RCR-032 single-process determinism (canonical global order) still holds unchanged; this RCR does
not touch `sim.rs`, `LoopbackTransport`, or any existing test. Per-node inbound is sorted by
canonical bytes (best-effort reproducibility for a fixed batch) — but correctness of committed
content does not depend on it (raft tolerates arrival-order/partial/retried delivery).

## HONEST SCOPE — proven vs still-OQ

- **Proven (real, tested):** cross-PROCESS networked consensus on one host over real length-framed
  TCP — separate OS processes, no shared memory, real connect/accept/reconnect/backoff and graceful
  shutdown; a quorum commits; committed outcome CONTENT is byte-identical to the in-process run.
- **NOT proven / deferred (recorded, NEVER quietly claimed):** true multi-HOST across machines (this
  is 127.0.0.1, separate processes on ONE host — the transport binds a *configurable* addr:port so
  it works across hosts *in principle*, but that is untested); hostile-network partition / latency /
  loss / reorder TIMING under a live network (these sockets deliver reliably; adversarial delivery
  stays the `sim.rs` filter/mangle model + OQ); TLS / mutual auth (mTLS is v2.0 debt, design OQ-7);
  a protocol-driven wire format (OQ-3 — the LE codec is internal framing); crash-durable raft state.
  The election timeout and reconnect backoff use real time in the DELIVERY layer only.
- **Anti-overread guard (permanent):** this score must stay worded **"cross-process networked
  consensus (one host, real TCP)"**. It must **never** be upgraded to "multi-host", "WAN",
  "partition/loss tolerant", or "network fault-tolerant".
