## HONEST LANGUAGE (load-bearing)

- **A read is a read.** `scan` replays the shard's WAL through the Query layer's
  `ShardProjection` (a pure fold of the committed prefix). It holds the store
  ONLY behind `&`, so no `Wal` write method (`append`/`install_snapshot`/
  `compact`) and no path to `Kernel::commit` is reachable from the scan — the
  Query layer stays "Writes: NOTHING" (OWN-001 / Layer Matrix).
- **In-process vs real-socket:** the `scan` verb reads through a SEPARATE,
  Arc-shared handle on the SAME durable store the Kernel commits to (exactly as
  the conformance reference `QueryProjection::from_store` and the I3 cluster
  rebuild do) — it observes every committed truth without ever holding a write
  handle. Over `--wal-dir` this is the real fsync-durable on-disk WAL; without
  it, the in-memory WAL. No new durability or replication claim is made.
- **Tenant isolation is structural (SHARD-001):** a foreign shard's record never
  enters the fold (`ShardProjection::apply` drops it), so a scan of shard A can
  never return B's records — proven, not asserted.
- **Determinism (HARD RULE 4):** the enumeration order is commit-offset order, a
  pure function of the committed trace (IDR-005, ORCH-003); two builds over the
  same prefix enumerate byte-identically. `scan` is a read and commits nothing,
  so it cannot be a source of nondeterminism in committed truth.

## Server-level proofs (all `respond()`-level, deterministic)

- **`scan` returns exactly the shard's committed set, in commit order:** a fresh
  session answers `scan 0`; after two commits it streams `scan 2 <cid0> <cid1>`
  in commit order; `scan bodies` appends `=<payload_hex>` to each id.
- **Tenant-isolated:** the same body committed into two shards yields two
  distinct truths; a scan of the default shard returns only its truth and never
  the other shard's — SHARD-001 at the fold level.
- **Malformed refused without echo:** `scan garbage` / `scan bodies extra` →
  `ERR bad-request`; a valid `id=` prefix is still echoed for correlation.
- **Survives the Kernel's death:** a scan over the recovered `--wal-dir` store
  (deterministic replay, ORCH-003) sees BOTH committed truths in order — total
  reconstruction survives a process kill+restart.
