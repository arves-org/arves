## HONEST LANGUAGE (load-bearing)

- **The Who rides INSIDE committed truth.** `commit-as` wraps the body in the
  SAME I5 attribution envelope the Control-Plane agent flow uses
  (`arves_control_plane::agents::encode_attributed`) and commits the envelope as
  ACS-001-addressed truth through the frozen gateway. The identity is in the WAL
  (the audit log, IDR-005), never in side metadata that could drift, and is
  recovered by `decode_attributed` over a `scan` (RCR-033).
- **Attribution changes identity, honestly.** An attributed commit is a DISTINCT
  truth from a plain commit of the same body (the envelope has a different ACS
  address) — proven (`assert_ne!`), not hidden. Re-attributing the same
  (agent, body) is idempotent (ORCH-004).
- **A CLAIMED Who (not enforced).** Mirroring the RCR-029/030 decoders, the
  `agent` token names whatever bytes the caller supplied; this verb does NOT
  verify the agent is registered committed truth (that structural gate is
  `propose_attributed`'s), nor that the CALLER is the agent (v2.0 debt #8). The
  envelope records the claim into the trail; it does not, by itself, prove it —
  stated in the code and here.
- **Backward compatible.** The plain-commit path is byte-unchanged; only the new
  `commit-as` verb wraps.

## Server-level proofs (all `respond()`-level, deterministic)

- **Round-trips and is queryable:** `commit-as 1220abcd 01 <body>` commits a
  distinct truth; a `scan bodies` yields the envelope, which
  `decode_attributed` decodes back to exactly `(agent=1220abcd, effect=body)`.
- **Distinct from plain + idempotent:** the attributed truth's ContentId differs
  from the plain commit of the same body; a re-`commit-as` answers
  `already-committed` with the SAME id (ORCH-004).
- **Unattributed commit still works:** the plain body committed alongside is NOT
  an attribution envelope (`decode_attributed` → `None`).
- **Malformed refused without echo:** bad agent hex / missing body / bad body
  hex / extra token → `ERR bad-request`; a valid `id=` still correlates.
