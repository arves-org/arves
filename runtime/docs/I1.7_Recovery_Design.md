# I1.7 — Recovery (Hardening) — Engineering Design

**Milestone:** I1.7 (sub-step of frozen Baseline `I1 Distributed Runtime`) — the
final storage-side step; completing it closes I1.
**Status:** Design in progress (phases 8–10 finalized from the adversarial hunt).
**Predecessor:** I1.6 Checkpoint Semantics (`tag: I1.6-checkpoint-semantics`)
**Governing:** ARVES Engineering Constitution; RT-001 for any interface change;
frozen specification is NOT modified.

---

## 0. One-line intent

Recovery already works on the happy path (I1.6: `load_snapshot → install_state →
replay(tail)`). I1.7 **proves it correct under crash/fault conditions** and
**hardens** the gaps that surface — above all: recovery must be **lossless or
loud**, never silently return partial truth.

## 1. Architecture Readiness Review

I1.6 delivered the recovery mechanism and green happy-path proofs (B12–B15). What
it did NOT do is systematically attack recovery under partial-write, partial-
compaction, corrupt-snapshot, torn-segment, and multi-shard fault combinations.
The constitution's mandatory tests (Failure-Injection · Recovery · Replay) and
"Critical Self-Review: destroy your own design" require that pass before I1 can
be declared complete. Ready.

## 2. Affected UCI node

**Persistence layer** (recovery scan `FileWalStore::open`, `compact`,
`install_snapshot`, `load_snapshot`, `replay_from`) and the **Kernel recovery
path** (`RefKernel::replay`). No new layer; no ownership change.

## 3. Specification mapping

- **ORCH-003 / IDR-003** — recovery is replay from the recorded trace (+ derived
  checkpoint), never recomputation. I1.7 must keep this exact under faults.
- **IDR-005** — append-only; compaction only deletes fully-covered sealed
  segments. Fault handling must never rewrite a survivor.
- **OWN-001 / ORCH-001 / PERSIST-001** — recovery reconstructs *the* single
  owner's truth from durable state; Persistence still owns no truth. A recovery
  that silently drops committed truth violates the spirit of OWN-001 (the owner's
  state must be faithfully restored) — hence "lossless or loud".
- **SHARD-001** — per-shard recovery; per-shard offsets.

## 4. Contract mapping

Expected to be **mostly additive**: a way for recovery to signal
*unrecoverable/lossy* state instead of returning silently (e.g. a
`RecoveryError` / a checked recover entry point) rather than the current
infallible `recover`. Any `Wal`/`Kernel` signature change will be justified under
**RT-001** (activating reserved durability semantics) and recorded in §D, exactly
as I1.6 was. No frozen spec change.

## 5. Invariant mapping

| Invariant | I1.7 obligation |
|---|---|
| ORCH-003 | Fault-injected recovery still reconstructs from the trace/snapshot; proven by tests. |
| IDR-005 | Partial-compaction recovery never rewrites survivors; only whole covered files are ever removed. |
| OWN-001 | Recovery is lossless for committed truth, or fails loudly (no silent partial truth). |
| ORCH-004 | Repeated/interrupted recovery is idempotent and deterministic. |

## 6. Ownership analysis

Unchanged from I1.6: Kernel is the sole truth owner and the only component that
serializes/deserializes snapshot state; Persistence stores opaque bytes and
manages segment/snapshot file lifecycle. I1.7 adds *detection* of unrecoverable
durable state, which is a Persistence-reports / Kernel-decides split (Persistence
reports "prefix compacted but no valid snapshot"; Kernel decides to fail rather
than fabricate/omit truth). No boundary moves.

## 7. IDR mapping

Still single-node: IDR-001..005 concern the distributed log. I1.7 does not
implement Raft/quorum/replication. It shapes recovery so that when replication
arrives (I2), a lost/corrupt local prefix can be *repaired from peers* rather
than silently dropped — the "loud failure" is the hook a replicated node will use
to trigger a snapshot/log transfer. No IDR implemented early; none violated.

## 8. Gap analysis — FROM THE ADVERSARIAL HUNT

The `i17-recovery-hunt` workflow ran **6 fault lenses**, each finding
adversarially verified (15 agents, 9 raw findings → **7 CONFIRMED, 2 REFUTED**).
Consolidated into four root defects:

| ID | Root defect | Severity | Hunt findings |
|----|-------------|----------|---------------|
| **A** | After compaction, a lost/corrupt prefix snapshot → recovery **silently** returns the surviving tail as if complete (silent data loss). | critical | #1, #6 |
| **B** | A corrupt/torn **interior (sealed) segment**: `decode_all` stops at the bad frame, `replay_from` returns a **gapped** trace; `open()` only inspects the last segment so `head` over-reports. | critical | #2, #3, #5 |
| **C** | When `snapshot.up_to+1 > recovered head` (a corrupt current segment truncated below the snapshot), recovery **panics** (`replay_from(...).expect`) even though the snapshot holds the full truth. | high | #4 |
| **D** | Crash in `install_snapshot`'s fsync→rename window leaves an orphan `*.snap.tmp` that is **never reclaimed** (unbounded disk leak). | low | #7 |

**REFUTED (recorded as considered-and-safe):** two "crash mid-compaction leaves
a hole" claims. Refuted because (1) `compact` unlinks segments in **ascending**
order and (2) the snapshot is installed and durable **before** any compaction,
so a partial compaction always leaves the snapshot + surviving segments
recoverable. The design's ordering guarantees held up under attack.

## 9. Engineering design (implemented)

Unifying principle enforced: **recovery is lossless or loud.**

- **A — loud refusal on lost compacted prefix.** `RefKernel::try_replay`: if
  `load_snapshot() == None` but `wal.earliest() > 0`, return
  `RecoveryError::CompactedPrefixWithoutSnapshot` instead of replaying only the
  tail.
- **B — completeness check in `replay_from`.** After decoding retained segments,
  verify the range `[offset, head)` is **dense and contiguous**; a gap or
  shortfall returns `WalError::Corruption { missing_offset, .. }`, which surfaces
  in the Kernel as `RecoveryError::Corruption`. Interior corruption can no longer
  produce a silent hole.
- **C — no panic when the snapshot covers everything.** `try_replay`: if
  `base > head`, the snapshot already holds every committed offset; skip the
  (empty) tail rather than calling `replay_from(base)` into an `OffsetOutOfRange`
  panic.
- **D — orphan sweep.** `FileWalStore::open` deletes stray `snap-*.snap.tmp`
  files on the recovery scan.

Recovery is now **fallible**: `try_recover -> Result<Self, RecoveryError>` is the
checked entry point; `recover` is a thin wrapper that **panics loudly** on error
(never returns a partial Kernel). Each defect has a fault-injection regression
test (kernel `tests/recovery.rs` R-A/R-B/R-C/R-OK; persistence `tests/recovery.rs`
interior-corruption / orphan-sweep / healthy-complete).

## 10. Critical self-review

The adversarial hunt **was** the independent critical self-review (6 lenses,
per-finding refutation). Residual, consciously-accepted single-node limits:

- **A corrupt sole snapshot after compaction is genuinely unrecoverable on a
  single node** — by design we now *detect and refuse* rather than repair. Repair
  requires a second copy, which arrives with replication (I2). The loud
  `RecoveryError` is exactly the hook a replicated node will use to fetch a
  snapshot/log from a peer.
- `open()` still only CRC-repairs the last segment's torn tail; interior
  corruption is caught at `replay_from` (loud), not silently — acceptable, and
  cheaper than CRC-scanning every segment on every open.
- Parent-directory fsync remains best-effort (std limitation; carried from I1.5).

No new residual **silent** failure modes remain in the audited surface.

## D. Decision Register — RT-001 interface evolution

Recovery gains a **checked failure mode** — Reference Runtime Interface Evolution
under RT-001 (activating the reserved "lossless or loud" durability semantics; no
frozen spec change, no layer added, no ownership moved).

| # | Change | Justification |
|---|--------|---------------|
| E1 | New `RecoveryError` enum + `RefKernel::try_recover` / `try_replay` (fallible); `recover`/`replay` become panicking wrappers | Recovery must be able to signal *unrecoverable* instead of returning partial truth. Additive; existing infallible call sites keep working (now loud on failure). |
| E2 | New `WalError::Corruption { shard, missing_offset, detail }` variant; `replay_from` now guarantees a dense/complete range or errors | Makes the durable store *self-checking* for interior corruption — Persistence detects durability failures (it owns durability), Kernel decides. |

Kernel remains the sole truth owner; Persistence still stores opaque bytes only.
**7/7 fault-injection proofs pass; 37/37 total behaviour tests pass; 0 warnings.**
