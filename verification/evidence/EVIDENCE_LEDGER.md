# ARVES Evidence Ledger

**The KPI artifact of the Standard Validation Era.** One place that answers, per claim:
*what is proven, to what Evidence Level, by whom, how to reproduce it, and how
independent the proof really is.* No property is "Done" — it holds an Evidence Level.
See `CERTIFICATION_PROGRAM.md` for the tier/grade definitions.

- **Section A** has two halves, and the difference is load-bearing:
  - **A.1 — probe-verified (drift-proof).** Regenerated by `evidence_probe.py`, which
    actually runs each suite and **owns** the Status + Metric cells. These cells cannot
    drift: the probe rewrites them, and `evidence_probe.py --check` fails the batch if the
    committed cell disagrees with a fresh run. Last probe: **12/12 PASS** (incl. the first
    end-to-end live L1 conformance pipeline — Information→Kernel→Query — from the real
    codec + Kernel + WAL, RCR-008/009/010; the ACS-003/004/005 **semantic differential**
    Rust-native vs Python; and the **capstone** organization-day, reproducible end-to-end).
  - **A.2 — asserted (NOT probe-run).** Product / P8 demonstration rows. Their reproduction
    command is real and was observed by hand, but **no probe re-runs them**, so their
    metrics **can drift** and are *not* covered by the drift gate. They are asserted, not
    machine-verified. Advancing one to A.1 means giving it a probe row + anchor.
- **Declared rows (Section B)** are earned/attacked by the destroy-offices and cite
  their artifact + status.
- **Independence (Section C)** is graded honestly: G0 self · G1 same-process · G2
  third-party. **The current maximum is G1.**

Regenerate: `python verification/evidence/evidence_probe.py`
(→ `evidence_ledger.tsv` + patches A.1 cells). Drift gate:
`python verification/evidence/evidence_probe.py --check`. Attack round: the
destroy-offices workflow.

> **The Era-3 exit gate is G2.** Three implementations in three languages
> (Rust / Python / TypeScript) now agree byte-for-byte — but all are at most **G1**
> (produced inside this program). ARVES is *reproduced within one program*, not yet
> independently validated. The cold TypeScript build (Kit-only, question-logging) is the
> sharpest G1 rehearsal to date and returned **YES-WITH-CAVEATS**: it built conformantly
> on the first run with zero back-and-forth, and its question-log exposed four Kit gaps —
> all now closed (§B). The only move left toward G2 is a genuine external team.
