# ARVES Evidence Ledger

**The KPI artifact of the Standard Validation Era.** One place that answers, per claim:
*what is proven, to what Evidence Level, by whom, how to reproduce it, and how
independent the proof really is.* No property is "Done" — it holds an Evidence Level.
See `CERTIFICATION_PROGRAM.md` for the tier/grade definitions.

- **Machine-verifiable rows (Section A)** are regenerated by `evidence_probe.py`, which
  actually runs each suite — they cannot drift from reality. Last probe: **6/6 PASS**.
- **Declared rows (Section B)** are earned/attacked by the destroy-offices and cite
  their artifact + status.
- **Independence (Section C)** is graded honestly: G0 self · G1 same-process · G2
  third-party. **The current maximum is G1.**

Regenerate: `python verification/evidence/evidence_probe.py`
(→ `evidence_ledger.tsv`). Attack round: the destroy-offices workflow.

## Section C — Independence ledger (the honesty gate)

| What | Grade | Why |
|------|-------|-----|
| Rust reference (arves-acs) | G0 self | the reference itself |
| Independent Python (ACS codec) | **G1 same-process** | fresh-context, Kit-only, Rust-source-forbidden — but produced inside this program |
| **Third-party runtime** | **G2 — NOT YET MET** | no stranger has built a passing runtime from the public Kit with no help |

> **The Era-3 exit gate is G2.** Everything above is at most G1. ARVES is not yet
> "independently validated" in the strong sense; it is *reproduced within one program*.
> Closing this gap — a public Kit + a third party who passes with no help — is the
> single highest-value evidence move remaining.
